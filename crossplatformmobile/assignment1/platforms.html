Mobile Application Platforms

Sometimes framework and platform seem interchangeable as terms. Some of these have been cited on different sites as frameworks and platforms both. I tried to differentiate by amount of features, or if something has its own IDE. Some sites also present that something is a platform only if it's completely stand-alone or a programming language like PHP, more on those in the techniques section.

    Xamarin is said to be a more complete development environment than PhoneGap and Titanium. One of the reasons is providing the possibility to test the app on thousands of virtual devices before launching. Uses C# [5]

    React Native uses Javascript and React. It’s also a powerful platform for building apps for android and iOS. For instagram, the features that were built with react native shared 90% of the code between platforms. Does not provide access to all the device’s functionality but you can build that yourself in native code. [5]

    Titanium is similar to React Native. It’s also written in Javascript and it also bridges native APIs to Javascript with it’s own APIs. Doesn’t rely on WebViews anymore so it provides a “native” look and feel. [5]

    PhoneGap/Cordova are probably the most well known. They use HTML, CSS and Javascript, run in WebView. PhoneGap provides native plugins that allow the use of the device’s functionality including accelerometer, camera, compass, file system, microphone, media, networks, notifications, geolocation, and storage. [5]

    You can also use MobiLoud Canvas for hybrid development. Canvas is offered as a service that builds, publishes and maintains the app. They have their own codebase for iOS and Android. Includes native elements such as a tab bar, a push notifications inbox, content preloading for your pages, caching, and offline support. The app is designed to rely on a remote web app or mobile site which you control – any change on your mobile site is immediately reflected in the app. Canvas can also be used to convert a web app into a hybrid app. [5]

    Chris Rickard provides Appcelerator as an alternative for cross-platform native development. For hybrid they also provide Trigger.IO, Ionic Framework and Telerik Platform.
